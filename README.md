Java Console Banking System (은행 계좌 관리 시스템)

이 프로젝트는 Java의 객체지향 프로그래밍(OOP) 핵심 개념인 인터페이스, 상속, 다형성을 활용하여 구현한 콘솔 기반의 은행 업무 시뮬레이션 프로그램입니다. 사용자는 일반 통장과 마이너스(신용) 통장을 개설할 수 있으며, 입금, 출금, 송금 및 계좌 관리 기능을 수행할 수 있습니다.

1. AccountOperationInterface.java

역할: 모든 종류의 계좌(일반, 마이너스)가 반드시 가져야 할 기본 기능을 정의합니다.

주요 메서드:

deposit(double amount): 입금

withdraw(double amount): 출금

transfer(Account yours, double amount): 계좌 이체

getBalance(): 잔액 확인

getAccountNo(): 계좌번호 확인

2. BankInterface.java

역할: 은행 시스템이 갖춰야 할 관리 기능을 정의합니다.

주요 메서드:

addAccount(Account account): 계좌 등록

getAccount(String accountNo): 계좌 조회

printBalance(): 전체 계좌 잔고 출력

gebalance(): 은행 총 자본금 확인

addTotalBalance / subTotalBalance: 은행 총 자산 갱신

3. Account.java (일반 통장)

역할: AccountOperationInterface를 구현한 기본 계좌 클래스입니다.

특징:

은행(Bank) 객체와 연결되어 있어, 입출금 시 은행의 총 자본금을 자동으로 갱신합니다.

출금 시 잔액이 부족하면 출금을 거부하는 기본 로직을 가집니다.

계좌 생성 시 은행의 관리 리스트에 자동으로 등록됩니다.

4. MinusAccount.java (마이너스 통장)

역할: Account 클래스를 상속(Extends) 받아 기능을 확장한 신용 계좌 클래스입니다.

특징:

Overriding (재정의): withdraw 메서드를 재정의하여, 잔액이 부족해도 설정된 한도(Credit Limit) 내에서 출금을 허용합니다.

신용 정보: 신용등급(rating)과 대출 한도(creditLimit) 필드를 추가로 보유합니다.

추가 기능: 잔액 확인 시 현재 잔액뿐만 아니라 한도와 출금 가능액을 계산하여 제공합니다.

5. Bank.java (은행 관리)

역할: BankInterface를 구현하며, 모든 계좌 객체를 관리하는 컨트롤 타워입니다.

특징:

ArrayList<Account>를 사용하여 다수의 계좌를 저장하고 관리합니다.

지급준비금 관리: 은행의 초기 자본금(1000.0)을 설정하고, 총 자본금이 준비금 이하로 떨어질 경우 경고 메시지를 출력합니다.

계좌 해지: deleteAccount 메서드를 통해 특정 계좌를 리스트에서 삭제합니다.

6. Main.java (실행 및 UI)

역할: 프로그램의 진입점(main 메서드 포함)이며 사용자 입력을 처리합니다.

특징:

Scanner를 이용해 사용자와 상호작용하며 메뉴(1~9번)를 제공합니다.

신용 평가 로직: 계좌 개설 시 입력된 신용점수에 따라 등급(1, 2등급)을 산정하거나 개설을 반려하는 로직을 수행합니다.

instanceof 연산자를 활용하여 계좌 타입(일반/마이너스)에 따른 차별화된 정보를 출력합니다.

3. 주요 기능 요약
계좌 개설: 신용점수에 따른 마이너스 통장 한도 차등 부여 (1등급: 1500, 2등급: 500).

입출금 및 송금: 실시간 잔액 변동 및 은행 총 자산 반영.

계좌 조회: 계좌 잔액, 신용 등급, 한도, 출금 가능액 조회.

계좌 해지: 기존 계좌 삭제 기능.

은행 관리: 은행 총 자본금 확인 및 지급준비금 모니터링.

4. 추가 기능
마이너스 통장 개설 시 입력된 신용 점수를 바탕으로 한도를 보여주는 기능.
